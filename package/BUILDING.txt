Building glinet-privacy and luci-app-glinet-privacy
=====================================================

One-command install on the router (replace USER/REPO):

  curl -fsSL https://raw.githubusercontent.com/USER/REPO/main/install.sh | \
    GLINET_PRIVACY_TARBALL_URL=https://github.com/USER/REPO/archive/refs/heads/main.tar.gz sh

Add LuCI UI:

  curl -fsSL ... | GLINET_PRIVACY_TARBALL_URL=... sh -s -- --with-luci

Default install.sh already runs opkg (WireGuard, Tor, dnsmasq-full, …), Tor torrc, cron,
telemetry — use --minimal to skip that.

From a git clone on the device:

  sh install.sh --with-luci

See install.sh header for GLINET_PRIVACY_SRC / GLINET_PRIVACY_GIT_URL.

Supported models (WAN defaults): see docs/devices.md (GL-XE300 Puli, GL-AXT1800 Slate AX, GL-AX1800 Flint).
IMEI rotation: same file, **IMEI rotation — legal use** section, before using --with-imei-*.
DNS: **Services → GL.iNet Privacy → Tor, DNS & telemetry** (`glinet_privacy.dns.dns_policy`); see `apply-dns-policy.sh` and `changes.md` 1.1.6.
Vendor VPN kill switch (GL.iNet `glvpn`): **Kill switch** page (`privacy.main.vendor_gl_vpn_killswitch`); see `docs/devices.md` and `changes.md` 1.1.7.

Versioning: edit package/version.mk and changes.md for each release; see changes.md.

Translations (LuCI): after editing `.lua` / `.htm` strings, run from the repo root:
  python tools/extract-luci-i18n-strings.py
  python tools/i18n-build-po-from-pot.py
Edit `package/luci-app-glinet-privacy/po/<lang>/glinet_privacy.po` (e.g. uk, de), then rebuild the ipk; `po2lmo` runs during `make package/luci-app-glinet-privacy/compile` (requires luci-base host). See `package/luci-app-glinet-privacy/po/README`.

CI: push a tag `v1.2.3` to trigger `.github/workflows/openwrt-packages.yml` and publish
`.ipk` files to a GitHub Release. PRs and pushes build artifacts only (no release). The workflow
pins the SDK image tag to `*-openwrt-24.10` and does not add a second luci feed (the SDK default
`feeds.conf` already defines `luci`; duplicating it breaks `./scripts/feeds update`).

---

These are standard OpenWrt packages. Copy this repository into your OpenWrt SDK or
buildroot `package/` tree (or add as a feed), then:

  ./scripts/feeds update -a
  ./scripts/feeds install glinet-privacy luci-app-glinet-privacy

Or from this repo:

  cp -a package/glinet-privacy /path/to/openwrt/package/
  cp -a package/luci-app-glinet-privacy /path/to/openwrt/package/

  make menuconfig
    Network  --->
      <*> glinet-privacy
    LuCI  --->
      Applications  --->
        <*> luci-app-glinet-privacy

  make package/glinet-privacy/compile V=s
  make package/luci-app-glinet-privacy/compile V=s

Install on the router:

  opkg install glinet-privacy_*.ipk luci-app-glinet-privacy_*.ipk

After install, open LuCI: Services → GL.iNet Privacy.

If the menu entry is missing, run: /etc/init.d/rpcd restart; /etc/init.d/uhttpd restart

ACL: ensure your LuCI login role can access the new app (rpcd acl
/usr/share/rpcd/acl.d/luci-app-glinet-privacy.json).

Firewall plugin: package postinst registers firewall.glinet_privacy pointing at
/usr/libexec/glinet-privacy/fw-plugin.sh (OpenWrt firewall include).
