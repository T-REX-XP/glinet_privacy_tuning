GL.iNet Privacy — supported on GL.iNet stock firmware only
===========================================================

install.sh refuses non-GL.iNet images. All steps below are automated unless you use --minimal.

One command (replace USER/REPO; one line avoids broken copy-paste with \):

  curl -fsSL https://raw.githubusercontent.com/USER/REPO/main/install.sh | GLINET_PRIVACY_TARBALL_URL=https://github.com/USER/REPO/archive/refs/heads/main.tar.gz sh -s --

Do not paste installer log lines back into the shell. GLINET_PRIVACY_TARBALL_URL must end in .tar.gz (source tree archive).

LuCI requires luci-lua-runtime; install.sh installs it with opkg and aborts if LuCI is enabled but the package cannot be installed. Default: Lua index menu; GLINET_PRIVACY_LUCI_MENU_JSON=1 installs menu.d JSON.

From a git clone on the router:

  sh install.sh

To remove (from the same clone, or set GLINET_PRIVACY_SRC to the repo root):

  sh remove.sh
  sh remove.sh --opkg    # if you installed luci-app-glinet-privacy / glinet-privacy via opkg

See remove.sh --help. Does not remove tor, wireguard-tools, etc., unless you use opkg for our ipks.

What install.sh does by default (not --minimal)
-----------------------------------------------
- Copies package/glinet-privacy/files to /, chmod scripts, installs /usr/share/glinet-privacy/version.mk
- Registers firewall.glinet_privacy → fw-plugin.sh; runs apply-device-profile.sh (WAN per device)
- opkg update + install: iptables, iptables-mod-*, wireguard-tools, kmod-wireguard, tor, tor-fw-helper,
  dnsmasq-full, ca-bundle, luci-lua-runtime when LuCI is enabled (best-effort if offline; LuCI step fails hard if missing)
- Tor: ensures %include /etc/tor/torrc.d/99-transparent.conf in /etc/tor/torrc; enables and starts tor
- Kill switch: enables /etc/init.d/privacy-killswitch; adds cron */1 for privacy-killswitch-watchdog.sh
- Telemetry: uci glinet_privacy.tel.block_domains=1 and disable_vendor_cloud=1; apply-telemetry.sh
  (dnsmasq.d black-holes for goodcloud.xyz / gldns.com, glconfig.cloud.enable=0, gl_cloud stop/disable);
  optional remove_cloud_packages removes gl-cloud via opkg; dhcp dnsmasq confdir /etc/dnsmasq.d when possible
- LuCI: copies luci-app-glinet-privacy into /usr/lib/lua/luci/... (Services → GL.iNet Privacy), unless --without-luci.
  Default: Lua index() only; GLINET_PRIVACY_LUCI_MENU_JSON=1 installs menu.d JSON + luci-use-menu-d (Lua index() skipped).
  After menu changes, delete /tmp/luci-indexcache* if stale.
- Restarts network, firewall, dnsmasq, rpcd, uhttpd

Optional flags
--------------
  --minimal           Files + firewall + device profile only (no opkg/Tor/cron/telemetry automation)
  --without-luci      Skip LuCI app files (default is to install them)
  --with-luci         Same as default (install LuCI app); kept for compatibility
  --with-imei-boot    Enable rotate_imei on boot (cellular; read docs/devices.md IMEI section)
  --with-imei-cron    Enable UCI IMEI cron (default every 6h); change interval in LuCI → IMEI rotation

VPN (WireGuard / OpenVPN)
-------------------------
Configure the client in the GL.iNet admin. Then set the WireGuard interface name in LuCI
(Services → GL.iNet Privacy → Kill switch) so the watchdog matches your tunnel.

Building ipk packages (OpenWrt buildroot)
-------------------------------------------
The repo ships **OpenWrt-style** packages under **`package/glinet-privacy/`** and
**`package/luci-app-glinet-privacy/`** (shared **`package/version.mk`**).

- Example **`feeds.conf`** lines: **`feeds.conf.example`** at the repository root
  (**`src-git` … `;subdir=package`** or **`src-link` …/package**).
- Details: **`package/OPENWRT-BUILD.txt`**.
- After **`./scripts/feeds update`** / **`install`**, enable **Network → glinet-privacy**
  and **LuCI → Applications → luci-app-glinet-privacy** in **`make menuconfig`**, then
  **`make package/glinet-privacy/compile package/luci-app-glinet-privacy/compile`**.

Notes
-----
- IMEI: legal use and operator/lab framing — **docs/devices.md** (IMEI rotation section); each run logs a notice unless `ROTATE_IMEI_SUPPRESS_LEGAL_LOG=1`.
- DNS: LuCI **Tor, DNS & telemetry** (`glinet_privacy.dns`) or **`apply-dns-policy.sh`** (`default` or `tor_dnsmasq`).
- Tor + VPN: use one routing path you understand; policy routing is outside this package.
- docs/devices.md — WAN defaults per model (Puli vs Slate AX).
